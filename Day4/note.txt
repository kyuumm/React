****俩种路由模式

各个主流框架的路由常用的路由模式有俩种，history模式和hash模式,
ReactRouter分别由 createBrowserRouter 和createHashRouter函数负责创建

路由模式    url表现       底层原理                     是否需要后端支持
history    url/login     history对象+pushState事件    需要
hash       url/#/login   监听 hashChange事件          不需要

import { createBrowserRouter, createHashRouter } from 'react-router-dom'

const router = createHashRouter([
])
//把createBrowserRouter替换成createHashRouter


用处:
让你的前端路由在“没有/不方便改后端配置”的情况下也能稳定工作，
尤其是刷新页面、直接粘贴链接访问时不会 404。

*****Mock
什么是数据Mock
在前后端分类的开发模式下，
前端可以在没有实际后端接口的支持下先进行接口数据的模拟，进行正常的业务功能开发
常见mock方式：1前端写假数据，2自研Mock平台，3json-server等工具
JsonServer实现步骤:
  1.项目中安装json-server
    npm i-D json-server（-D？使用-D（开发依赖）：
                  包会被安装但只在开发环境使用
                  会被记录在package.json的devDependencies中
                  在生产环境构建时不会被安装
                  适用于构建工具、测试框架、开发服务器等只在开发阶段需要的工具）
  2.准备一个json文件
  3.添加启动命令
    "server":"json-server ./server/data.json --port 8888"
  4.访问接口进行测试


****<Outlet />
是 React Router 提供的一个组件，它在嵌套路由中起到"占位符"的作用，用于渲染子路由的组件。
具体来说：
当你配置了嵌套路由时，一级路由组件中需要使用 <Outlet /> 来指定二级路由组件的渲染位置
当访问某个二级路由时，二级路由的组件内容会渲染在一级组件中 <Outlet /> 所在的位置
如果没有匹配到任何二级路由，<Outlet /> 不会渲染任何内容


****主题定制
      定制方案:
      1.全局定制
      整个应用范围内的组件都生效
          :root:root {
              --adm-color-primary: #a062d4;
          }
          //
      2.局部定制
      只在某些元素内部的组件生效
          .purple-theme {
              --adm-color-primary: #a062d4;
          }

      使用：      
      <Button color="primary">test global</Button>
      <div className="part">
        <Button color="primary">test part</Button>
      </div>
